# CDP采购系统开发质量规范（Kiro Steering）

> 参赛团队使用方法：将此文件复制到你的项目 `.kiro/steering/quality-standards.md`
> Kiro 会在开发过程中自动遵守以下规范。

---

## 使用说明

1. 在你的项目根目录创建 `.kiro/steering/` 目录
2. 将此文件保存为 `.kiro/steering/quality-standards.md`
3. Kiro 会自动读取并在所有代码生成中遵守这些规范

---

以下是 steering 文件内容，直接复制即可：

```markdown
# 质量规范 — CDP采购系统AI竞赛

## 安全性规范（必须遵守）

### 1. 登录认证
- 所有API接口必须经过身份认证中间件
- 使用JWT或Session进行身份验证
- 登录接口使用手机号+密码

### 2. 账号锁定
- 连续输入错误密码5次后锁定账号30分钟
- 记录失败次数和锁定时间

### 3. 会话超时
- JWT过期时间设置为30分钟
- 或Session maxAge设置为30分钟

### 4. 密码加密
- 密码必须使用bcrypt加密存储，saltRounds >= 10
- 禁止明文存储密码
- 接口返回数据中禁止包含密码字段

### 5. SQL注入防护
- 禁止字符串拼接SQL语句
- 必须使用参数化查询或ORM
- 使用Prisma/TypeORM/Sequelize等ORM框架

### 6. XSS防护
- 禁止使用innerHTML直接插入用户输入
- 使用框架自带的转义机制（React/Vue默认安全）
- 如需渲染HTML，必须使用DOMPurify等库过滤

### 7. 权限隔离
- 供应商只能查看本企业数据，查询必须带tenantId（租户ID）过滤
- 普通采购员只能查看分配给自己的PR，查询带assigned_buyer_id过滤
- 业务人员端路由和菜单中不展示RFQ相关入口
- Admin/采购经理可查看全量数据
- 租户隔离推荐使用中间件统一注入tenantId，避免每个接口手动过滤

### 8. 报价保密
- 开标前（RFQ状态非BID_OPENED/COMPLETED），报价金额对所有用户不可见
- 查询报价接口必须检查RFQ开标状态

## 性能规范（必须遵守）

### 1. 列表分页
- 所有列表查询接口必须支持分页参数（page, pageSize）
- 默认pageSize为20，最大不超过100
- 返回总数total用于前端分页组件

### 2. 防重复提交
- 前端表单提交按钮必须有loading状态
- 提交过程中按钮disabled，防止重复点击
- 关键接口后端可加幂等键校验

### 3. 数据库索引
- 常用查询字段（status、supplier_id、buyer_id、created_at）必须建索引
- 外键字段必须建索引
- 唯一约束字段（手机号、PR号、供应商ID）使用unique索引

### 4. 查询优化
- 禁止使用SELECT *，必须指定需要的字段
- 列表查询不返回大文本字段（如备注、附件内容）
- 热点数据考虑使用Redis缓存

### 5. 并发控制
- 供应商信息、联系人等共享数据使用乐观锁（version字段）
- 更新时WHERE条件包含version，更新失败提示"数据已被修改"

### 6. 文件上传限制
- 单文件大小限制100MB
- 禁止上传可执行文件：.exe .bat .cmd .sh .js .vbs .msi .apk
- 证件附件仅允许：PDF、JPG、PNG
- 报价/合同附件允许：PDF、Word、Excel、JPG、PNG、ZIP、RAR

## 测试规范（必须遵守）

### 1. 测试覆盖
- 每个核心模块至少5条测试用例
- 必须覆盖正常流程和至少2种异常场景
- 测试文件按模块组织，文件名包含模块关键词

### 2. 测试报告
- 配置测试报告生成命令（HTML格式）
- 报告输出到项目约定目录
- 提交前确保所有测试通过

### 3. 测试命名
- 测试文件名包含业务模块关键词：supplier、pr、rfq、contract、po、payment
- 测试用例描述清晰，包含场景说明

## 编码规范

### 接口设计
- RESTful风格，资源名使用复数
- 统一响应格式：{ code, message, data }
- 错误响应包含明确的错误信息

### 错误处理
- 不可逆操作（删除、取消、停用）必须弹出二次确认
- 表单提交前进行客户端校验，显示具体错误提示
- 操作完成后提供明确的成功/失败反馈

### 编号规则
- 供应商ID：VD + 4位自增（VD 0001）
- PR号：PR-YYYYMM-5位自增（PR-202605-00001）
- RFQ号：RFQ-YYYYMM-5位自增
- 合同号：CA-YYYYMM-5位自增
- PO号：PO-YYYYMM-5位自增
```
