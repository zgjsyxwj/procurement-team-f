# 实体关系用例分析文档

## 用例 1：最简单闭环

场景：1 个 PR 创建 1 个 RFQ，1 个供应商提交 1 份 Quote，最终生成 1 份 Contract。

```text
PR-001
  -> RFQ-001
    -> Round-1
      -> Quote-001 by Supplier-A
        -> Quote Version-001
    -> Contract-001 with Supplier-A
```

关键点：

- RFQ.pr_id = PR-001。
- Quote.rfq_id = RFQ-001。
- Quote.round_id = Round-1。
- Quote.supplier_id = Supplier-A。
- Quote Version.quote_id = Quote-001。
- Contract.pr_id = PR-001。
- Contract.rfq_id = RFQ-001。
- Contract.supplier_id = Supplier-A。

## 用例 2：一个 PR，多个供应商报价，选一个供应商签合同

场景：1 个 PR 创建 1 个 RFQ，邀请 3 个供应商，3 个供应商都报价，最终选择 Supplier-A 签合同。

```text
PR-001
  -> RFQ-001
    -> RFQ Supplier: Supplier-A
    -> RFQ Supplier: Supplier-B
    -> RFQ Supplier: Supplier-C
    -> Round-1
      -> Quote-A1
        -> Quote Version-A1-V1
      -> Quote-B1
        -> Quote Version-B1-V1
      -> Quote-C1
        -> Quote Version-C1-V1
    -> Contract-001 with Supplier-A
```

关键点：

- RFQ Supplier 表示供应商被邀请或参与。
- Quote 表示供应商实际报价。
- Quote Version 表示该报价的具体提交版本。
- Contract.supplier_id 记录最终签约供应商。

## 用例 3：供应商被邀请但未报价

场景：RFQ 邀请 3 个供应商，只有 2 个供应商提交报价。

```text
RFQ-001
  -> RFQ Supplier: Supplier-A
  -> RFQ Supplier: Supplier-B
  -> RFQ Supplier: Supplier-C

Round-1
  -> Quote-A1
    -> Quote Version-A1-V1
  -> Quote-B1
    -> Quote Version-B1-V1
```

关键点：

- Supplier-C 有 RFQ Supplier 记录。
- Supplier-C 没有 Quote 记录。
- 不能只通过 Quote 判断 RFQ 参与供应商。

## 用例 4：一个 PR，一个 RFQ，多轮报价

场景：第一轮报价后开启第二轮议价，最终使用第二轮结果生成合同。

```text
PR-001
  -> RFQ-001
    -> Round-1
      -> Quote-A1
        -> Quote Version-A1-V1
      -> Quote-B1
        -> Quote Version-B1-V1
    -> Round-2
      -> Quote-A2
        -> Quote Version-A2-V1
      -> Quote-B2
        -> Quote Version-B2-V1
    -> Contract-001 with Supplier-B
```

关键点：

- RFQ 与 RFQ Round 是 1:N。
- 每轮 Quote 独立保存。
- 每个 Quote 可有多个 Quote Version。
- 第二轮 Quote 不覆盖第一轮 Quote。

## 用例 5：一个 PR 创建多个 RFQ

场景：1 个 PR 被拆成 2 个 RFQ 分别询价，最终形成 2 份合同。

```text
PR-001
  -> RFQ-001
    -> Quote-A1
      -> Quote Version-A1-V1
    -> Contract-001 with Supplier-A

  -> RFQ-002
    -> Quote-B1
      -> Quote Version-B1-V1
    -> Contract-002 with Supplier-B
```

关键点：

- PR 与 RFQ 是 1:N。
- PR 与 Contract 是 1:N。
- 每个 Contract 都需要关联自己的 RFQ。

## 用例 6：多个 PR 合并成一个 PR Bundle 后询价

场景：PR-001 和 PR-002 合并成一个 PR Bundle，再基于合集创建 RFQ。

```text
PR Bundle-001
  -> PR Bundle Item -> PR-001
  -> PR Bundle Item -> PR-002

PR Bundle-001
  -> RFQ-001
    -> Quote-A1
      -> Quote Version-A1-V1
    -> Contract-001 with Supplier-A
```

关键点：

- PR Bundle Item.bundle_id = PR Bundle-001。
- PR Bundle Item.pr_id = PR-001 或 PR-002。
- RFQ.source_type = PR_BUNDLE。
- RFQ.bundle_id = PR Bundle-001。
- RFQ.pr_id 为空。

## 用例 7：PR Bundle 下创建多个 RFQ

场景：多个 PR 合并后，合集下拆成多个 RFQ 处理。

```text
PR Bundle-001
  -> PR-001
  -> PR-002
  -> PR-003

PR Bundle-001
  -> RFQ-001
    -> Contract-001 with Supplier-A

  -> RFQ-002
    -> Contract-002 with Supplier-B
```

关键点：

- PR Bundle 与 RFQ 是 1:N。
- PR Bundle 与 Contract 是 1:N。
- Contract.bundle_id 关联 PR Bundle。

## 用例 8：单一来源，采购员代录报价

场景：PR 使用单一来源策略，只选择一个供应商，由采购员代录 Quote。

```text
PR-001
  -> RFQ-001 策略：单一来源
    -> RFQ Supplier: Supplier-A
    -> Round-1
      -> Quote-A1 source=PROXY
        -> Quote Version-A1-V1
    -> Contract-001 with Supplier-A
```

关键点：

- 单一来源仍然需要 RFQ。
- 单一来源仍然需要 Quote。
- 单一来源仍然需要 Quote Version。
- Quote.source = PROXY 表示采购员代录。

## 用例 9：报价被退回后重新提交

场景：供应商第一次报价版本被退回，修改后重新提交新版本。

```text
RFQ-001
  -> Round-1
    -> Quote-A1
      -> Quote Version-A1-V1 状态：已退回
      -> Quote Version-A1-V2 状态：已报价
```

关键点：

- RFQ、Round、Supplier 关系不变。
- Quote 关系不变。
- 需要能识别当前有效 Quote Version。
- 被退回和重新提交需要保留版本记录。

## 用例 10：一个 RFQ 生成多份合同

场景：一个 RFQ 核价完成后，分批创建多份合同。

```text
PR-001
  -> RFQ-001
    -> Quote-A1
      -> Quote Version-A1-V1
    -> Contract-001 with Supplier-A
    -> Contract-002 with Supplier-A
```

关键点：

- RFQ 与 Contract 是 1:N。
- PR 与 Contract 是 1:N。
- 多份合同可以关联同一个 RFQ 和同一个 Supplier。

## 用例 11：同一个供应商参与多个 RFQ

场景：Supplier-A 同时参与多个询价单。

```text
Supplier-A
  -> RFQ Supplier -> RFQ-001
  -> RFQ Supplier -> RFQ-002

Supplier-A
  -> Quote-A1 for RFQ-001
    -> Quote Version-A1-V1
  -> Quote-A2 for RFQ-002
    -> Quote Version-A2-V1
```

关键点：

- Supplier 与 RFQ Supplier 是 1:N。
- Supplier 与 Quote 是 1:N。
- Quote 与 Quote Version 是 1:N。
- 供应商参与记录和报价记录是两类实体。

## 用例 12：单据都有明细，但明细不跨单据关联

场景：PR、RFQ、Quote Version、Contract 都有自己的明细，但明细之间不建立外键。

```text
PR-001
  -> PR Line-001
  -> PR Line-002

RFQ-001
  -> RFQ Line-001
  -> RFQ Line-002

Quote-001
  -> Quote Version-001
    -> Quote Line-001
    -> Quote Line-002

Contract-001
  -> Contract Line-001
  -> Contract Line-002
```

关键点：

- PR Line.pr_id 关联 PR.id。
- RFQ Line.rfq_id 关联 RFQ.id。
- Quote Version.quote_id 关联 Quote.id。
- Quote Line.quote_version_id 关联 Quote Version.id。
- Contract Line.contract_id 关联 Contract.id。
- PR Line、RFQ Line、Quote Line、Contract Line 之间不互相关联。
