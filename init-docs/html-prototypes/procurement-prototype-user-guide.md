# 采购系统高保真原型使用说明

适用对象：采购部门  
适用范围：用于评审和体验采购系统 HTML 高保真原型，帮助采购同事理解页面入口、按钮功能、单据状态和业务流转。

---

## 1. 原型入口

推荐使用按流程拆分的原型入口：

`/Users/qing/Documents/caigou/03-业务演示与原型/html-prototypes/procurement-html-prototype-flow-scenarios/index.html`

## 3. 角色说明

### 3.0 业务人员

业务人员负责在采购系统中发起采购申请单（PR），查看本人 PR 的审批、询价、合同、PO 和付款进度，并在合同签署完成后发起 PO。

主要菜单：

| 菜单 | 用途 |
| --- | --- |
| 采购申请单 | 创建 PR、查看本人 PR 和下游进度 |
| 合同管理 | 查看本人 PR 关联合同，合同完成后创建 PO |
| PO管理 | 查看本人发起 PO 的Worklife BPM 审批、台账同步和付款状态 |

### 3.1 采购员

采购员负责日常采购执行，包括供应商维护、PR查看、询价、核价、合同创建、PO和付款跟踪。

主要菜单：

| 菜单 | 用途 |
| --- | --- |
| 工作台 | 查看待办和快捷入口 |
| 供应商管理 | 创建供应商、维护供应商联系人和证件 |
| 审批中心 | 查看我发起的、需要我审批的、审批完成的事项 |
| 采购申请单 | 查看Worklife BPM 审批系统分配给自己的 PR，并从 PR 发起询价；MVP 1.0 历史方案中也可查看 RPA 同步 PR |
| 询价单管理 | 创建询价单、查看 RFQ；线下寻源独立入口已删除 |
| 合同管理 | 从已完成 RFQ 创建合同，并跟踪审批和签署 |
| 履约与付款 | 查看 PO 和付款状态 |
| 邮件日志 | 查看邀请、询价、审批等邮件发送情况 |

### 3.2 采购经理

采购经理拥有采购员权限，并可查看全量数据、处理审批、查看报表和维护基础配置。

主要菜单：

| 菜单 | 用途 |
| --- | --- |
| 经理工作台 | 查看采购金额、流程效率、风险事项 |
| 审批中心 | 审批 RFQ 核价等事项 |
| 采购申请单 | 查看全量 PR |
| 询价单查询 | 查询全量 RFQ |
| 合同管理 | 查看合同状态 |
| 履约与付款 | 查看 PO 和付款状态 |
| 采购分析 | 查看支出、流程、供应商、风险报表 |
| 数据设置 | 维护采购员账号、字段设置、表单设置 |

### 3.3 供应商

供应商只查看本企业相关信息，包括企业信息、收到的询价单、合同。

主要菜单：

| 菜单 | 用途 |
| --- | --- |
| 企业信息 | 维护企业基础信息、银行信息、证件资料 |
| 我的询价单 | 查看收到的询价单，提交或查看报价 |
| 合同管理 | 查看待签署和已签署合同 |
| 邮件日志 | 查看系统邮件记录 |

说明：供应商端「企业信息」和「我的询价单」是两个独立入口。「我的询价单」列表和详情中不再放「完善企业信息」按钮。

## 4. 主流程总览

采购系统 MVP 的主流程如下：

```text
业务人员在采购系统发起 PR
  ↓
Worklife BPM 审批系统审批并分配采购员
  ↓
采购员从PR创建询价单
  ↓
供应商在线报价 / 采购员代录报价
  ↓
报价截止后采购员核价
  ↓
采购员提交核价审批
  ↓
采购经理审批通过
  ↓
采购员创建合同
  ↓
外部审批推送合同审批
  ↓
OA审批结果回传
  ↓
电子签署或线下签署上传归档
  ↓
合同完成
  ↓
业务人员发起 PO 并查看付款状态
```

## 6. 采购申请单 PR

### 6.1 入口

业务人员端 → 我的采购申请 / 创建采购申请单

采购员端 → 采购申请单

MVP 2.0 当前目标方案中，PR 由业务人员在采购系统内发起，提交Worklife BPM 审批系统审批。审批通过后，Worklife BPM 审批系统会回传审批结果并分配采购员；只有被分配采购员和采购经理能在采购端看到该 PR。MVP 1.0 历史方案中，PR 来自 Worklife BPM审批通过后的 RPA 同步结果，作为历史兼容方案保留。

### 6.2 列表按钮

| 按钮 | 功能 |
| --- | --- |
| 创建采购申请单 | 业务人员填写并提交 PR |
| 提交审批 | 将业务人员创建的 PR 提交Worklife BPM 审批系统 |
| 模拟审批通过并分配采购员 | 原型演示用，表示Worklife BPM 审批系统回传审批通过和采购员分配结果 |
| 跟踪详情 | 进入业务人员 PR 跟踪详情，查看审批、分配采购员、合同、PO和付款状态；业务人员端不展示 RFQ |
| 查看详情 | 进入采购端 PR 详情 |
| 创建询价管理单 | 基于当前 PR 创建 RFQ |
| 创建PR合集 | 将多张 PR 合并成一张 PR 合集 |

### 6.3 PR详情页签

业务人员端的 PR 详情以「PR基本信息」和「下游单据」为主，不展示采购员操作按钮。页面展示审批单号、分配采购员、合同、PO、付款状态和已付款总额；业务人员端下游单据区不展示 RFQ。

| 页签 | 内容 |
| --- | --- |
| PR概览 | 查看 PR 基本信息和需求内容 |
| 包含PR | 如果是 PR 合集，查看合集下的原始 PR |
| RFQ | 仅采购员/采购经理端展示，用于查看该 PR 关联的询价单；业务人员端不展示 |
| 合同 | 查看该 PR 关联的合同 |
| PO与付款 | 查看该 PR 关联的多个 PO 和付款汇总 |

### 6.4 PR 状态流转

```text
草稿
  ↓ 提交Worklife BPM 审批
审批中
  ↓ 审批驳回
审批驳回
  ↓ 修改后重新提交
审批中
  ↓ 审批通过并分配采购员
待询价
  ↓ 创建询价单
询价中
  ↓ 报价截止并开标
待核价
  ↓ 提交核价审批
核价审批中
  ↓ 审批驳回
核价退回
  ↓ 调整后重新提交
核价审批中
  ↓ 审批通过
待建合同
  ↓ 合同签署归档
采购流程完成
```

PR 主状态只表示采购流程进度。“采购流程完成”不代表付款完成；PO审批状态、台账同步状态、付款状态会继续在列表和详情中展示。

### 6.5 PR合集规则

| 场景 | 规则 |
| --- | --- |
| PR合集未生成RFQ | 可以在详情中移除某条原始 PR |
| PR合集已生成RFQ | 不允许移除合集内 PR |
| PR合集创建RFQ | 和普通 PR 一样进入询价流程 |

## 7. 创建询价单

### 7.1 入口

采购员端 → 采购申请单 → 创建询价管理单  
或  
采购员端 → 询价单管理 → 创建询价单

### 7.2 创建步骤

| 步骤 | 内容 | 关键操作 |
| --- | --- | --- |
| 1. 选择采购申请单 | 搜索并选择 PR 或 PR合集 | 支持按 PR号/需求内容、部门、PR类型筛选 |
| 2. 项目基本信息 | 填写 RFQ 项目信息，并维护报价内容设置 | 支持添加、编辑、删除报价明细行 |
| 3. 报价安排 | 设置报价开始、截止、开标时间 | 开标时间必须晚于报价截止时间 |
| 4. 参与供应商 | 选择供应商和联系人 | 支持按名称、类型、证件状态筛选 |
| 5. 确认发布 | 最终确认并发布 | 保存草稿或保存并发布 |

### 7.3 页面按钮

| 按钮 | 功能 |
| --- | --- |
| 下一步 | 进入下一步配置 |
| 上一步 | 返回上一步 |
| 添加报价明细行 | 在项目基本信息中新增一行标准报价内容 |
| 保存草稿 | RFQ 保存为草稿，不通知供应商 |
| 保存并发布 | RFQ 进入报价中，供应商可在供应商端看到 |

### 7.4 RFQ 状态流转

```text
草稿
  ↓ 保存并发布
报价中
  ↓ 报价截止
报价截止 / 已开标
  ↓ 采购员进入核价
核价中
  ↓ 提交核价审批
审批中
  ↓ 采购经理审批通过
已完成
  ↓ 可创建合同
待建合同
```

审批驳回时：

```text
审批中
  ↓ 审批驳回
已驳回 / 核价退回
  ↓ 采购员补充后重新提交
审批中
```

## 8. 采购员代录报价

### 8.1 入口

采购员端 → 询价单管理 → 创建询价单 → 选择单一来源/定向采购/续约 → RFQ详情 → 代录供应商报价

线下寻源独立入口已删除。原线下报价场景统一并入单一来源、定向采购、续约采购策略，由采购员代录报价。

### 8.2 页面按钮

| 按钮 | 功能 |
| --- | --- |
| 代录供应商报价 | 录入供应商提供的报价 |
| 编辑 | 修改尚未提交核价的代录报价 |
| 保存并进入代录报价 | RFQ 保存并进入代录报价状态 |
| 关闭报价 | 系统记录报价截止流水并立即进入已开标流程 |

### 8.3 代录供应商报价需要填写

| 字段 | 说明 |
| --- | --- |
| 供应商名称 | 本次代录报价供应商 |
| 联系人 | 供应商联系人 |
| 电话/邮箱 | 联系方式 |
| 物料/服务名称、数量、单价、小计、服务/交货天数 | 报价明细 |
| 报价附件 | 必须上传供应商报价凭证 |
| 备注 | 记录补充说明 |

### 8.4 状态流转

```text
创建询价单并选择单一来源/定向采购/续约
  ↓ 保存并进入代录报价
代录至少一条供应商报价并上传附件
  ↓
核价中
```

代录报价不会给供应商生成在线报价待办，也不会要求供应商在门户中提交报价。

## 9. 供应商报价

### 9.1 入口

供应商端 → 我的询价单

### 9.2 列表按钮

| 按钮 | 功能 |
| --- | --- |
| 详情 | 查看询价单详情、采购需求、报价记录 |
| 去报价 | 进入报价填写页面 |
| 查看 | 查看已提交或已完成的询价 |

### 9.3 报价规则

| 规则 | 说明 |
| --- | --- |
| 报价截止前仅可提交一次 | 提交后供应商不能自行修改 |
| 采购员退回后可重新提交 | 退回原因会显示在询价详情中 |
| 报价需要上传附件 | 附件作为报价凭证 |
| 我的询价单不引导完善企业信息 | 企业信息维护从「企业信息」菜单单独进入 |

### 9.4 供应商报价状态

```text
待报价
  ↓ 供应商提交
已报价
  ↓ 采购员退回
已退回
  ↓ 供应商重新提交
已报价
  ↓ RFQ结束并核价
已完成
```

## 10. 核价与改价

### 10.1 入口

采购员端 → 询价单管理 → 询价单详情 → 进入核价流程  
或  
流程场景首页 → 核价与审批

### 10.2 核价页面内容

| 区域 | 说明 |
| --- | --- |
| 核价进度 | 展示当前核价所处阶段 |
| 卡片统计 | 展示报价、预算差异、供应商数量等核价概览 |
| 供应商报价明细 | 查看各供应商报价 |
| 改价记录 | 查看采购员对最终成交价的调整记录 |
| 核价结论 | 展示最终核价结果 |

### 10.3 页面按钮

| 按钮 | 功能 |
| --- | --- |
| 报价复核 | 进入报价明细复核 |
| 改价 | 对某条供应商报价明细录入最终成交价 |
| 提交审批 | 将核价结果提交给采购经理 |
| 返回核价总览 | 返回核价首页 |

### 10.4 改价规则

改价只影响最终成交价，不覆盖供应商原始报价。

改价会记录：

| 内容 | 说明 |
| --- | --- |
| 单价和小计 | 供应商原始报价；改价不会覆盖该原始报价 |
| 调整后最终价 | 采购员核价后的成交价 |
| 改价原因 | 为什么调整 |
| 操作人和时间 | 方便追溯 |

### 10.5 核价状态流转

```text
核价中
  ↓ 采购员完成核价并提交
审批中
  ↓ 采购经理审批通过
已完成
  ↓
可创建合同
```

如果审批驳回：

```text
审批中
  ↓ 驳回
核价退回
  ↓ 采购员补充或调整
审批中
```

## 12. 合同管理

### 12.1 入口

采购员端 → 合同管理

合同创建优先选择 PR 或 PR 合集；如果该 PR 关联多个已审批完成 RFQ，再选择对应 RFQ 和供应商报价。

### 12.2 合同列表按钮

| 按钮 | 功能 |
| --- | --- |
| 创建合同 | 选择 PR/PR 合集，必要时选择已完成 RFQ，创建合同申请 |
| 详情 | 查看合同详情、审批和签署状态 |

### 12.3 创建合同页面

创建合同时需要先选择 PR 或 PR 合集；如果存在多个已审批完成 RFQ，再选择对应 RFQ。系统会带出：

| 内容 | 说明 |
| --- | --- |
| PR | 合同来源采购申请 |
| RFQ | 合同来源询价单 |
| 中标供应商 | 核价后的供应商 |
| 合同明细 | 中标物料或服务 |
| 金额 | 合同金额数字 |
| 货币 | CNY、USD、EUR 等 |

页面按钮：

| 按钮 | 功能 |
| --- | --- |
| 保存合同草稿 | 合同进入合同草稿状态 |
| 生成合同申请并推送审批 | 合同进入 外部审批推送中 |
| 取消 | 返回合同管理 |

### 12.4 合同状态流转

```text
合同草稿
  ↓ 推送Worklife BPM审批
外部审批推送中
  ↓ 模拟Worklife BPM接收成功
Worklife BPM审批中
  ↓ 模拟Worklife BPM审批通过
Worklife BPM审批通过
  ↓ 选择电子签
待签署
  ↓ 发起电子签署
电子签署中
  ↓ 模拟签署完成并归档
合同完成
```

线下签署路径：

```text
Worklife BPM审批通过
  ↓ 上传线下签署文档并归档
合同完成
```

或：

```text
待签署
  ↓ 改为线下签署并上传归档
合同完成
```

### 12.5 合同详情按钮

| 当前状态 | 可点击按钮 | 点击后状态变化 |
| --- | --- | --- |
| 合同草稿 | 推送Worklife BPM审批 | 变为 外部审批推送中 |
| 外部审批推送中 | 模拟Worklife BPM接收成功 | 变为 Worklife BPM审批中 |
| Worklife BPM审批中 | 模拟Worklife BPM审批通过 | 变为 Worklife BPM审批通过 |
| Worklife BPM审批通过 | 选择电子签 | 变为 待签署 |
| Worklife BPM审批通过 | 上传线下签署文档并归档 | 填写归档信息后变为 合同完成 |
| 待签署 | 发起电子签署 | 变为 电子签署中 |
| 待签署 | 改为线下签署并上传归档 | 填写归档信息后变为 合同完成 |
| 电子签署中 | 模拟签署完成并归档 | 变为 合同完成 |
| 合同完成 | 查看PO | 进入 PO 详情 |

### 12.6 线下签署归档需要填写

| 字段 | 说明 |
| --- | --- |
| 线下合同编号 | 线下签署合同编号 |
| 签署完成日期 | 双方完成签署日期 |
| 采购方签署人 | 采购方签署人 |
| 供应商签署人 | 供应商签署人 |
| 归档文件 | 双方盖章签署完成的文件 |
| 文件校验 | 是否已核对双方盖章签字 |
| 归档备注 | 补充说明 |

确认归档后，合同状态进入「合同完成」，并回写 PR 的合同状态。

## 17. 常用状态速查

### 17.1 PR 状态

| 状态 | 含义 |
| --- | --- |
| 草稿 | 业务人员已创建，尚未提交Worklife BPM 审批 |
| 审批中 | 已提交Worklife BPM 审批系统 |
| 审批驳回 | Worklife BPM 审批驳回，业务人员可修改重提 |
| 待询价 | PR 已审批通过并分配采购员，尚未进入询价 |
| 询价中 | 已创建 RFQ，采购内部询报价处理中 |
| 待核价 | 报价截止并开标，等待采购员核价 |
| 核价审批中 | 核价结果已提交采购经理审批 |
| 核价退回 | 采购经理驳回核价结果，采购员可调整后重提 |
| 待建合同 | 核价审批通过，等待合同创建或签署归档 |
| 采购流程完成 | 合同已签署归档，采购流程完成；不代表付款完成 |
| 已取消 | PR 已取消或关闭 |

### 17.2 RFQ 状态

| 状态 | 含义 |
| --- | --- |
| 草稿 | 未发布 |
| 报价中 | 供应商可报价 |
| 报价截止 | 已到报价截止时间 |
| 核价中 | 采购员核价中 |
| 审批中 | 采购经理审批中 |
| 已完成 | 核价审批通过 |
| 已驳回 | 审批或报价被退回 |

### 17.3 合同状态

| 状态 | 含义 |
| --- | --- |
| 合同草稿 | 合同申请已保存但未推送 OA |
| 外部审批推送中 | 系统正在将合同申请推送给 Worklife BPM |
| Worklife BPM审批中 | Worklife BPM 正在审批合同 |
| Worklife BPM审批通过 | Worklife BPM 审批已通过，等待签署 |
| 待签署 | 等待发起电子签或线下归档 |
| 电子签署中 | 电子签流程进行中 |
| 合同完成 | 合同已签署并归档 |

### 17.4 PO与付款状态

| 状态 | 含义 |
| --- | --- |
| 未生成 | 尚未发起 PO |
| 审批中 | 业务人员已提交 PO 至Worklife BPM 审批系统 |
| 审批通过 | Worklife BPM 审批系统已通过，等待或正在同步台账 |
| 审批驳回 | Worklife BPM 审批系统驳回，业务人员可修改重提 |
| 同步中 / 已同步 / 同步失败 | 台账系统对接状态，作为“台账同步状态”查看，不与 PO审批状态混用 |
| 未付款 | 尚未付款 |
| 待付款 | 等待付款 |
| 付款中 | 正在付款处理 |
| 部分付款 | 已支付部分金额 |
| 付款完成 | 已完成付款 |
| 付款失败 | 付款失败，需要处理 |

## 18. 建议演示路径

建议采购部门评审时按以下顺序演示：

1. 打开流程场景版原型。
2. 进入「供应商创建与邀请」，演示创建供应商和发送邀请。
3. 进入「创建询价单」，演示从 `PR-202605-001` 创建 RFQ。
4. 进入「供应商报价」，演示供应商报价和退回原因。
5. 进入「采购员代录报价」，演示单一来源/定向采购/续约下的报价录入和附件上传。
6. 进入「核价与审批」，演示改价和提交审批。
7. 切换采购经理端「审批中心」，演示审批通过。
8. 进入「合同创建、Worklife BPM审批与签署」，演示合同创建、OA流转、电子签或线下归档。
9. 进入「PO履约与付款闭环」，演示 PO 详情和付款状态。
10. 进入「数据设置」，演示字段库和表单配置。
