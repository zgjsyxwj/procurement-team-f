# 采购系统 HTML 原型业务测试脚本

## 1. 这份文档怎么用

这份文档是给业务部门做高保真原型评审用的，不是给研发跑自动化测试用的。

业务同事可以按下面的步骤打开 HTML 原型，然后照着指定的 PR、RFQ、合同或 PO 数据点击，检查页面流程是否符合真实采购业务。

原型文件：

`/Users/qing/Documents/caigou/03-业务演示与原型/html-prototypes/procurement-html-prototype-flow-scenarios/index.html`

如果希望按业务流程逐个演示，可以打开流程场景版原型：

`/Users/qing/Documents/caigou/03-业务演示与原型/html-prototypes/procurement-html-prototype-flow-scenarios/index.html`

流程场景版会把「业务人员发起PR」「供应商创建」「创建询价单」「供应商报价」「采购员代录报价」「核价审批」「合同」「PO付款」「数据设置」拆成独立入口，适合会议演示时直接按场景点击。

采购部门阅读版使用说明：

`/Users/qing/Documents/caigou/03-业务演示与原型/procurement-prototype-user-guide.md`

打开后先选择角色：

| --- | --- |
| 业务人员发起 PR、查看本人 PR、合同完成后发起 PO | 业务人员端 |
| 采购员工作台、供应商、PR、询价、核价、合同、PO | 采购员端 |
| 采购经理工作台、审批、全量PR、数据设置 | 采购经理端 |
| 企业信息、我的询价单、供应商合同 | 供应商端 |

## 2. 推荐测试顺序

建议业务部门按这个顺序走一遍，比较像真实采购全流程：

1. 业务人员创建 PR 并模拟第三方审批通过、分配采购员
2. 采购员看工作台
3. 被分配采购员查看 PR
4. 采购员从 PR 创建询价单
5. 采购员走单一来源/定向采购/续约的代录报价
6. 采购员做核价和改价
7. 采购经理审批核价
8. 采购员创建合同
9. 合同审批、签署、归档
10. 业务人员基于已完成合同创建 PO，并模拟第三方审批通过
11. 查看 PO 和付款回传
12. 供应商完善企业信息、查看询价和合同
13. 采购经理查看报表和数据设置

## 4. 采购员端测试

### 4.1 采购员工作台

测试目的：确认采购员首页只展示真正需要处理的事项。

操作步骤：

1. 打开 HTML 原型。
2. 选择「采购员端」。
3. 查看首页顶部统计。
4. 查看「快捷操作」区域。
5. 点击「创建供应商」「创建询价单」「创建合同申请」。

重点检查：

- 不应该有「待建合同」统计卡片。
- 询价单统计里不应该有「草稿」和「已完成」。
- 快捷操作里应该有「创建供应商」「创建询价单」「创建合同申请」。
- 每个快捷入口都可以点击进入对应页面。

### 4.2 供应商详情和联系人邀请

测试目的：确认采购员可以直接维护供应商联系人，并可发送邀请邮件。

建议数据：

`上海博夷信息技术有限公司`

操作步骤：

1. 采购员端点击左侧「供应商管理」。
2. 点击供应商 `上海博夷信息技术有限公司` 进入详情。
3. 查看页面顶部按钮。
4. 切换到「联系人」Tab。
5. 点击「新增联系人」。
6. 返回联系人列表，点击某个联系人行内的「发送邀请邮件」。

重点检查：

- 页面上不应该有「添加管理关系」按钮。
- 采购员可以新增或编辑联系人。
- 新增或编辑联系人不需要审批。
- 联系人列表里有「发送邀请邮件」按钮。
- 点击发送邀请邮件后有确认反馈。

### 4.3 采购申请单列表

测试目的：确认 PR 列表展示的是状态汇总，不展示具体 PO 编号。

建议数据：

- `PR-202605-001`
- `PR-202605-014`

操作步骤：

1. 采购员端点击「采购申请单」。
2. 查看列表字段。
3. 找到 `PR-202605-014`。
4. 查看该行的 PO审批状态、台账同步状态、付款状态、已付款总额。

重点检查：

- 列表要有「已付款总额」。
- 列表不应该有「最终PO」字段。
- 列表不应该直接展示具体 PO 编号。
- 列表可以点击「查看详情」。
- 列表可以点击「创建询价管理单」。

### 4.4 PR 详情中查看多个 PO

测试目的：确认一个 PR 可以关联多个 PO，PO 明细放在 PR 详情里。

建议数据：

`PR-202605-014`，客服耳机采购。

操作步骤：

1. 采购员端进入「采购申请单」。
2. 点击 `PR-202605-014`。
3. 进入 PR 详情后，点击「PO与付款」Tab。
4. 查看关联 PO 列表。
5. 点击 `PO-202605-001` 或 `PO-202605-009`。
6. 在 PO 详情里点击关联 PR 号。

重点检查：

- PR 详情里可以看到该 PR 下的多个 PO。
- 每个 PO 都展示 PO编号、关联合同、供应商、PO金额、PO审批状态、台账同步状态、付款状态、已付款金额。
- PO 编号可以点击进入 PO 详情。
- PO 详情里的关联 PR 号可以点击回到 PR 详情。

### 4.5 从 PR 创建询价单

测试目的：确认采购员可以从 PR 发起询价管理单。

建议数据：

`PR-202605-001`，2026 年品牌活动礼品采购。

操作步骤：

1. 采购员端进入「采购申请单」。
2. 找到 `PR-202605-001`。
3. 点击该行「创建询价管理单」。
4. 进入创建询价单页面。
5. 第 1 步使用筛选条件搜索：在 PR号/需求内容中输入 `PR-202605-001`，或按部门筛选「市场部」。
6. 第 2 步查看项目基本信息。
7. 第 4 步查看参与供应商。

重点检查：

- PR 列表可以直接发起创建询价管理单。
- PR 详情顶部也可以发起创建询价管理单。
- 创建询价单第 1 步可以通过 PR号/需求内容、部门、PR类型筛选选择采购申请单。
- 创建询价单第 2 步「项目基本信息」下方应展示「报价内容设置」，支持新增、编辑、删除报价明细行。
- 第 3 步「报价安排」只设置采购策略、可报价时间和开标时间。
- 第 4 步供应商选择里不应该再出现线下寻源内容。
- 供应商支持按名称、类型、证件状态筛选。
- 选供应商后可以选择该供应商下的联系人。

### 4.6 PR 合集

测试目的：确认多张 PR 可以合并成 PR 合集，并且 PR 合集也可以进入询价流程。

建议数据：

- 已生成 RFQ 的合集：`PRG-202605-001`
- 尚未生成 RFQ 的合集：`PRG-202605-002`

操作步骤：

1. 采购员端进入「采购申请单」。
2. 点击「创建PR合集」，查看弹窗内容。
3. 打开 `PRG-202605-001`。
4. 点击「包含PR」Tab。
5. 再打开 `PRG-202605-002`。
6. 点击「包含PR」Tab。

重点检查：

- PR 合集可以作为询价来源。
- `PRG-202605-001` 已经生成 RFQ，里面的原始 PR 不允许移除。
- `PRG-202605-002` 尚未生成 RFQ，应该可以看到移除入口。
- 移除原始 PR 时需要填写原因。

### 4.7 采购员代录报价

测试目的：确认线下寻源独立入口已删除；单一来源、定向采购、续约通过「创建询价单」进入采购员代录报价流程。

建议数据：

`PR-202605-003`，上海办公室绿植租摆服务。

操作步骤：

1. 采购员端点击「询价单管理」。
2. 点击「创建询价单」。
3. 在第 1 步选择 `PR-202605-003`。
4. 在第 3 步报价安排中选择「单一来源」「定向采购」或「续约」。
5. 第 4 步仅保留一家供应商。
6. 确认页点击「保存并进入代录报价」。
7. 在 RFQ 详情中点击「代录供应商报价」，填写供应商、联系人、报价明细和报价附件。
8. 保存报价后进入统一核价流程。

重点检查：

- 询价单管理列表顶部不再展示「线下寻源」同级按钮。
- 代录报价由采购策略触发，不作为 RFQ 类型展示。
- 「代录供应商报价」按钮可以打开弹窗或录入页面。
- 报价附件在代录报价弹窗或页面里上传或填写。
- 页面不应该有单独的「附件与核价准备」模块。
- 代录报价保存后进入统一核价流程。

### 4.8 核价和改价

测试目的：确认核价页面符合 MVP 范围，只做报价复核、行级改价和提交审批。

建议数据：

`RFQ-202605-001`，2026 年品牌活动礼品采购。

操作步骤：

1. 采购员端点击「询价单管理」。
2. 打开 `RFQ-202605-001`。
3. 进入核价页面。
4. 查看供应商报价明细。
5. 在任意报价行点击「改价」。
6. 在弹窗里查看单价、小计、调整后最终成交价、改价原因。
7. 点击确认。
8. 点击「提交采购经理审批」。
9. 在提交确认弹窗中确认核价总额、最终成交价和核价说明后提交。

重点检查：

- 改价按钮是在报价明细的每一行上。
- 页面不应该有「核价项」字段。
- 页面不应该有「审批前检查」模块。
- 页面不应该有「预览审批摘要」按钮，但应有提交确认弹窗。
- 改价不会覆盖供应商原始报价，只作为最终成交价快照。

### 4.9 创建合同和合同流转

测试目的：确认合同从已审批完成 RFQ 创建，并且包含 外部审批推送 的中间状态。

建议数据：

创建合同时选择 `RFQ-202605-001`。

操作步骤：

1. 采购员端点击「合同管理」。
2. 点击「创建合同」。
3. 在可选 RFQ 里选择 `RFQ-202605-001`。
4. 查看系统带出的 PR、RFQ、中标供应商、金额和货币。
5. 点击「生成合同申请并推送审批」。
6. 进入合同详情，查看当前合同状态。
7. 点击状态旁边的「?」。
8. 模拟「外部审批推送中」到「Worklife BPM审批中」再到「Worklife BPM审批通过」。
9. 模拟电子签流程，或在 Worklife BPM审批通过后点击「上传线下签署文档并归档」。
10. 在线下归档弹窗中查看线下合同编号、签署完成日期、双方签署人和归档文件字段。

重点检查：

- 合同管理只展示已创建合同，不展示待建合同虚拟数据。
- 创建合同必须选择已审批完成的 RFQ。
- 合同状态里有「外部审批推送中」。
- 外部流程说明用状态旁边的「?」展示。
- 合同金额在列表和详情中拆分为「金额」和「货币」。
- 线下签署必须上传双方签署文档并确认归档后，合同才进入完成状态。
- 合同完成后可进入履约与付款。

## 6. 供应商端测试

### 6.1 首次进入企业信息

测试目的：确认供应商端没有工作台首页，企业信息维护作为独立入口展示。

操作步骤：

1. 打开 HTML 原型。
2. 选择「供应商端」。
3. 查看默认进入的页面。
4. 查看企业基础信息、银行信息、证件资料。
5. 点击「上传/更新证件」。

重点检查：

- 供应商端没有工作台首页。
- 默认进入「企业信息」页面。
- 企业信息和证件资料在同一个维护流程中。
- 企业信息维护和询价报价各自独立；报价入口在「我的询价单」中，不在企业信息页中处理。

### 6.2 查看报价被退回原因

测试目的：确认供应商知道为什么报价被采购退回。

建议数据：

供应商端 `RFQ-202605-003`，办公室绿植租摆服务。

操作步骤：

1. 供应商端点击「我的询价单」。
2. 找到报价状态为「已退回」的询价单。
3. 进入询价单详情。
4. 查看「我的报价记录」。

重点检查：

- 页面显示退回原因。
- 供应商可以看到需要补充的内容。
- 「我的询价单」列表和详情不展示「完善企业信息」按钮。
- 企业信息维护与询价报价各自独立，供应商从「企业信息」菜单单独维护资料。

## 7. 给业务演示时的推荐话术

可以按下面这条主线讲：

1. “当前目标方案里，业务人员在采购系统发起 PR，PR 走第三方审批；审批通过后会分配采购员。”
2. “只有被分配采购员和采购经理能在采购端看到这笔 PR；外部审批 同步 PR 是 MVP 1.0 历史方案，会保留兼容说明。”
3. “采购员可以从被分配 PR 创建询价，也可以把多张 PR 合并成 PR 合集后统一询价。”
4. “询价统一从创建询价单进入；询比价走供应商在线报价，单一来源、定向采购、续约走采购员代录报价，不再有线下寻源独立入口。”
5. “报价完成后进入核价，采购员可以在报价行上做改价，但不会覆盖供应商原始报价。”
6. “核价提交后，采购经理统一在审批中心处理。”
7. “审批通过后，采购员在合同管理里选择已完成 RFQ 创建合同。”
8. “合同创建后先由 外部审批推送，OA 审批结果再回传采购系统。”
9. “合同签署归档后，业务人员在合同上创建 PO，PO 走第三方审批；审批通过后同步台账系统付款。”
10. “一个 PR 可以有多个 PO，所以 PR 列表只展示状态和已付款总额，具体 PO 要到 PR 详情里看。”
11. “供应商端企业信息维护和询价报价是两个独立入口；我的询价单里不再放完善企业信息按钮，合同里金额和货币也拆开展示。”
